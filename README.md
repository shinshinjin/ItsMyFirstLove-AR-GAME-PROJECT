## 게임 소개

**연애는 처음이라서...** 는 모바일 AR 환경에서 사용자의 실제 위치를 기반으로 플레이할 수 있는 메타버스 게임입니다.  
게임의 주요 플레이 위치는 **천호 지역**으로 고정되어 있으며, 사용자는 실제 공간을 이동하면서 게임 내 캐릭터와 상호작용하고 이벤트를 진행할 수 있습니다.

게임은 크게 두 가지 모드로 구성되어 있습니다.

<br/>

### 스토리 모드

스토리 모드는 정해진 경로를 따라 이동하며 특정 위치에서만 발생하는 이벤트를 체험하는 모드입니다.  
스테이지별로 이동 경로가 다르기 때문에, 각 스테이지에 맞는 경로 데이터를 불러와 캐릭터가 해당 경로를 따라 이동하도록 구현했습니다.

<br/>

### 산책 모드

산책 모드는 특정 경로에 구애받지 않고 플레이할 수 있는 모드입니다.  
사용자는 자유롭게 이동하면서 캐릭터와 상호작용하고, 호감도를 올리는 방식으로 게임을 진행할 수 있습니다.

<br/>

## 담당 업무

### 1. NavMeshAgent 기반 캐릭터 이동 구현

캐릭터가 스테이지별 경로를 따라 이동할 수 있도록 **NavMeshAgent 기반 이동 시스템**을 구현했습니다.

스테이지마다 이동 경로가 다르기 때문에 `GameManager`에서 `CurrentStageIndex` 값을 관리하도록 구성했습니다.  
사용자가 특정 스테이지를 선택하면 `OnClick` 이벤트를 통해 해당 스테이지의 경로 데이터를 불러오고, 캐릭터가 해당 경로를 따라 이동하도록 처리했습니다.

또한 `RenewalLocation()` 함수를 활용하여 캐릭터가 경로를 갱신하며 이동하도록 구현했고, `GameManager`의 `StageEnd()` 함수를 통해 마지막 경로 도달 여부를 판별하여 스테이지 종료 처리를 진행했습니다.

<br/>

### 2. 거리 기반 캐릭터 상태 업데이트

AR 환경에서 플레이어와 캐릭터 간의 상호작용 상태를 안정적으로 판단하기 위해, 콜라이더 대신 거리 기반 판별 방식을 사용했습니다.

캐릭터 위치와 플레이어 위치의 차이를 `sqrMagnitude`로 계산하여, 두 오브젝트 간의 거리에 따라 캐릭터의 상태를 업데이트하도록 구현했습니다.

<br/>

#### 콜라이더를 사용하지 않은 이유

AR 환경에서 `OnTriggerStay`와 `OnTriggerExit`가 정상적으로 작동하지 않는 문제가 발생했습니다.  
이로 인해 콜라이더 기반 감지 방식 대신, 플레이어와 캐릭터의 위치 차이를 직접 계산하는 방식으로 변경했습니다.

이 방식은 AR 환경에서 트리거 이벤트가 불안정하게 동작하는 문제를 줄이고, 거리 기반으로 캐릭터 상태를 보다 안정적으로 제어할 수 있다는 장점이 있었습니다.

<br/>

### 3. 미니맵 구현

`Render Texture`를 활용하여 미니맵을 구현했습니다.

미니맵의 전체화면 표시와 최소화 기능은 Canvas의 `sortingOrder`를 조정하는 방식으로 구현했습니다.  
이를 통해 사용자가 필요에 따라 미니맵을 크게 확인하거나, 플레이 화면을 더 넓게 볼 수 있도록 UI 전환 기능을 구성했습니다.

<br/>

### 4. VPS Effect 구현

코루틴을 활용하여 특정 위치에서 오브젝트가 이동하고 멈추는 연출을 구현했습니다.

AR 공간 내에서 위치 기반 이벤트가 자연스럽게 보이도록 오브젝트 이동 타이밍과 정지 시점을 제어했으며, 이를 통해 특정 장소에서만 발생하는 AR 이벤트 연출을 구성했습니다.

<br/>

### 5. NavMeshBuilder를 활용한 실시간 NavMesh 최적화

AR 환경에서는 Google Map 데이터를 실시간으로 불러오기 때문에, 일반적인 방식처럼 게임 시작 시 한 번만 NavMesh를 Bake하는 방식으로는 대응하기 어려웠습니다.

이를 해결하기 위해 **NavMeshBuilder**를 활용하여 Google Map과 함께 실시간으로 NavMesh를 Bake하는 구조를 구현했습니다.

다만 맵 전체를 실시간으로 Bake하면 연산 비용이 크고 프레임 드랍이 발생할 수 있기 때문에, NavMesh를 Bake할 영역을 제한하는 방식으로 최적화를 진행했습니다.

<br/>

### 6. NavMeshBuildSource 관리 구조 구현

실시간으로 변경되는 건물 및 오브젝트 정보를 관리하기 위해 `NavMeshBuildSource` 리스트를 구성했습니다.

맵이 생성될 때 건물 오브젝트에 `NavMeshSourceTag` 스크립트를 추가하고, 해당 오브젝트가 NavMeshBuilder의 영역 안에 들어왔을 때만 Bake 대상에 포함되도록 구현했습니다.

반대로 NavMeshBuilder 영역 밖으로 벗어난 오브젝트는 Bake 대상에서 제외하여, 불필요한 연산을 줄이고 프레임 드랍을 최소화했습니다.

<br/>

## 담당 구현 상세

### NavMeshAgent 이동 구조

스테이지 인덱스에 따라 경로를 불러오고, `NavMeshAgent`를 통해 캐릭터가 해당 경로를 따라 이동하도록 구현했습니다.

주요 사용 요소는 다음과 같습니다.

- `CurrentStageIndex`
- `RenewalLocation()`
- `StageEnd()`

스테이지별로 경로가 다르기 때문에 현재 진행 중인 스테이지 인덱스를 기준으로 경로를 불러오도록 구성했습니다.  
캐릭터는 `NavMeshAgent`를 통해 목표 지점으로 이동하며, 마지막 경로에 도달하면 스테이지 종료 여부를 판단합니다.

<br/>

### 거리 기반 상호작용 판별

플레이어와 캐릭터의 위치 차이를 `sqrMagnitude`로 계산하여 상태를 업데이트했습니다.

AR 환경에서 Trigger 이벤트가 불안정하게 작동하는 문제를 해결하기 위해, 콜라이더가 아닌 거리 계산 방식을 사용했습니다.  
이를 통해 플레이어와 캐릭터 사이의 거리에 따라 캐릭터 상태를 안정적으로 변경할 수 있었습니다.

<br/>

### 실시간 NavMesh Bake 최적화

Google Map 데이터를 기반으로 생성되는 오브젝트를 실시간으로 NavMesh Bake 대상에 포함하거나 제외하는 구조를 구현했습니다.

주요 사용 요소는 다음과 같습니다.

- `NavMeshBuilder`
- `NavMeshBuildSource`
- `NavMeshSourceTag`

Google Map 데이터를 기반으로 생성되는 오브젝트를 실시간으로 NavMesh Bake 대상에 포함하거나 제외하는 구조를 구현했습니다.  
NavMeshBuilder의 영역 안에 들어온 오브젝트만 Bake되도록 하여 전체 맵을 Bake할 때 발생하는 성능 저하를 줄였습니다.

<br/>

## 트러블슈팅

### AR 환경에서 Trigger 이벤트가 정상적으로 동작하지 않는 문제

#### 문제

AR 환경에서 `OnTriggerStay`, `OnTriggerExit` 이벤트가 의도한 대로 동작하지 않는 문제가 발생했습니다.  
이로 인해 캐릭터와 플레이어의 상호작용 상태가 불안정하게 변경되는 문제가 있었습니다.

<br/>

#### 해결

콜라이더 기반 감지 방식을 사용하지 않고, 플레이어와 캐릭터의 위치 차이를 직접 계산하는 방식으로 변경했습니다.  
`sqrMagnitude`를 활용하여 거리 계산 비용을 줄이고, 특정 거리 안에 들어왔는지 여부를 기준으로 캐릭터 상태를 업데이트했습니다.

<br/>

#### 결과

AR 환경에서도 플레이어와 캐릭터 간 거리 판별이 안정적으로 동작했으며, Trigger 이벤트에 의존하지 않아 상호작용 로직의 안정성이 향상되었습니다.

<br/>

### 실시간 NavMesh Bake로 인한 프레임 드랍 문제

#### 문제

Google Map 기반 오브젝트를 실시간으로 불러오는 구조이기 때문에, 전체 맵을 대상으로 NavMesh를 Bake할 경우 연산 비용이 커지고 프레임 드랍이 발생할 수 있었습니다.

<br/>

#### 해결

NavMeshBuilder를 활용하여 Bake 영역을 제한했습니다.  
또한 `NavMeshBuildSource` 리스트를 만들어 실시간으로 변경되는 건물 오브젝트 정보를 관리하고, `NavMeshSourceTag`를 통해 Bake 대상 오브젝트를 구분했습니다.

<br/>

#### 결과

필요한 영역만 NavMesh Bake 대상에 포함되도록 하여 불필요한 연산을 줄였고, AR 환경에서 발생할 수 있는 프레임 드랍을 최소화했습니다.

<br/>

## 배운 점

이번 프로젝트를 통해 모바일 AR 환경에서 위치 기반 게임을 구현할 때 고려해야 할 점을 경험할 수 있었습니다.

특히 일반적인 3D 게임 환경과 달리 AR 환경에서는 콜라이더 이벤트가 안정적으로 동작하지 않을 수 있다는 점을 확인했고, 이를 거리 기반 판별 방식으로 대체하여 문제를 해결했습니다.

또한 Google Map 기반 오브젝트가 실시간으로 생성되는 환경에서 NavMesh를 어떻게 관리해야 하는지 고민했고, NavMeshBuilder와 NavMeshBuildSource를 활용하여 필요한 영역만 Bake하는 최적화 구조를 구현했습니다.

이를 통해 AR 환경에서의 위치 기반 캐릭터 이동, 실시간 NavMesh 처리, 성능 최적화에 대한 경험을 쌓을 수 있었습니다.
